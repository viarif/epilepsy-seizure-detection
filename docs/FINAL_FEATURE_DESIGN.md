# 最终特征提取方案（基于实际需求的重新设计）

## 🎯 设计原则

### 核心约束
1. **硬件友好** - 16-bit量化，适合FPGA/MCU实现
2. **计算高效** - 避免复杂运算，优先简单可靠的特征
3. **实用优先** - 不追求论文的全部细节，抓住核心有效特征
4. **可调试性** - 特征要容易理解和验证

### 重新审视"功率比值"

**功率比值真的难实现吗？**

让我算一下：
```python
# 假设已经有频段功率
delta_power = 5.2
theta_power = 3.8

# 比值计算
ratio = delta_power / theta_power  # 一次除法！
# 或者用对数（更稳定）
log_ratio = delta_power - theta_power  # 一次减法！（因为log(a/b) = log(a) - log(b)）
```

**结论**: 
- 如果功率已经取对数 → 比值就是**减法**！
- 硬件实现：1次减法 vs 1次加法，复杂度相同
- **功率比值不难实现，反而是最简单的特征之一**

### 核心思路
1. **只用真正有效的特征** - 参考3篇论文的共识

3. **分层设计** - 基础层（必须）+ 增强层（可选）

### 基础层：28个特征（必须实现）

#### 1. 时域特征（7个）- 仅最重要的通道
**从FZ-CZ提取**（中线参考通道，论文证明最稳定）

```python
1. Standard Deviation (标准差) - 信号强度
2. Skewness (偏度) - 波形不对称性
3. Hjorth Activity - 信号能量
4. Hjorth Mobility - 频率特性
5. Hjorth Complexity - 波形复杂度
6. Peak-to-Peak (峰峰值) - 振幅范围
7. Zero Crossing Rate (过零率) - 频率粗估计
```

**为什么只用1个通道？**
- 时域特征在不同通道间相似度高
- 论文1用单通道就达到90%准确率
- 减少冗余，节省计算

**为什么去掉Mean？**
- EEG信号已去均值/高通滤波
- 均值 ≈ 0，无区分度

---

#### 2. 频域特征（15个）- 3通道 × 5频段
**从所有3通道提取**（T7-P7, T8-P8, FZ-CZ）

**频段选择**（5个关键频段）:
```python
1. δ (delta): 4-8 Hz      - 深度睡眠，脑损伤
2. θ (theta): 8-13 Hz     - 困倦，情绪
3. β (beta): 13-30 Hz     - 清醒，焦虑
4. γ_low: 30-50 Hz        - 认知，癫痫特征 ← 论文2重点
5. Broadband: 4-50 Hz     - 总功率
```

**每通道5个频段 → 3通道 × 5 = 15个频域特征**

**为什么不用>50Hz？**
- 50Hz是工频干扰
- >50Hz主要是肌电噪声（EMG）
- 论文2只用到48Hz
- 避免引入噪声

**为什么加Broadband（总功率）？**
- 归一化参考
- 可用于计算相对功率
- 论文3强调相对功率重要

---

#### 3. 功率比值（6个）- 关键通道的关键比值
**从FZ-CZ提取**（最稳定的通道）

```python
1. δ/θ  (Delta-Theta Ratio)
2. θ/β  (Theta-Beta Ratio) 
3. (δ+θ)/β  (Slow/Fast Ratio)
4. β/γ_low  (Beta-Gamma Ratio)
5. δ/Broadband  (Delta相对功率)
6. γ_low/Broadband  (Gamma相对功率)
```

**为什么只用1个通道的比值？**
- 论文3证明：单通道的好比值 > 多通道的差比值
- 比值特征已经很强，不需要冗余
- FZ-CZ最稳定，最适合提取比值

**为什么这6个比值？**
- δ/θ, θ/β: 经典脑电比值
- (δ+θ)/β: 论文3提到的低频/高频比
- β/γ_low: 论文2强调γ_low重要
- 相对功率: 论文3强调其重要性

**计算成本？**
- 如果功率用对数: ratio = log(P1) - log(P2) → **一次减法**
- 比加法复杂度还低！

---

### 基础层总结
```
时域: 7个  (1通道 FZ-CZ)
频域: 15个 (3通道 × 5频段)
比值: 6个  (1通道 FZ-CZ)
────────────────────
总计: 28个特征
```

**计算成本**:
- 1次FFT（3通道共享）
- 7次简单统计
- 6次减法（比值）
- **总计: O(n log n)** - 被FFT主导

## 🎯 为什么这是最优方案？

### 1. 硬件友好
- **28个特征** → 小矩阵，适合16-bit量化
- **主要运算**: FFT + 简单统计 + 减法
- **无复杂操作**: 无高阶矩、无非线性变换
- **内存占用低**: 28 × 1841窗口 × 2字节 ≈ 100KB

### 2. 抓住核心
- **Hjorth参数**: 论文1证明最有效 ✓
- **低γ频段**: 论文2证明重要 ✓
- **功率比值**: 论文3证明最强区分度 ✓
- **多通道频域**: 论文2证明通道选择重要 ✓

### 3. 避免冗余
- **不重复提取**: 相似特征只在1个通道提取
- **不盲目覆盖**: 不是每个论文特征都要用
- **针对性强**: 每个特征都有明确目的

### 4. 易于实现和调试
- **特征少**: 28个特征，容易可视化
- **计算简单**: 没有复杂的数学运算
- **可解释性**: 每个特征都有物理意义

### 5. 可扩展
- **基础层充分**: 28个特征足够训练
- **增强层备用**: 需要时加入小波
- **灵活调整**: 可根据实验结果增减

## 📝 代码实现预览

```python
def extract_features_final(window, sfreq=256):
    """
    最终方案特征提取
    
    Args:
        window: [3, 1024] - 3通道，4秒窗口
        
    Returns:
        features: [28] or [40] - 特征向量
    """
    features = []
    
    # 1. 时域特征 (7个) - 仅FZ-CZ通道
    time_features = extract_time_domain(window[2])  # FZ-CZ
    features.extend(time_features)  # 7个
    
    # 2. 频域特征 (15个) - 所有3通道
    freq_features = extract_frequency_domain(window, sfreq)
    features.extend(freq_features)  # 15个
    
    # 3. 功率比值 (6个) - 仅FZ-CZ通道
    ratio_features = compute_power_ratios(freq_features[10:15])  # FZ-CZ的5个频段
    features.extend(ratio_features)  # 6个
    
    # 4. (可选) 小波特征 (12个)
    # wavelet_features = extract_wavelet(window)
    # features.extend(wavelet_features)
    
    return np.array(features)  # 28个 (or 40个)
```
